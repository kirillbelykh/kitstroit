# Архитектура

## Состав

```text
Browser
  ├─ /              React static build
  ├─ /admin         React admin
  ├─ /api           Async FastAPI
  ├─ /media         Bundled launch media
  └─ /uploads       Admin uploads on VPS volume
                       │
                       ├─ PostgreSQL
                       └─ Telegram Bot API
```

Один Docker Compose запускает reverse proxy, frontend, backend и PostgreSQL. Отдельные микросервисы, брокер сообщений и Kubernetes не нужны.

## Backend

- Python 3.12, `uv`.
- Async FastAPI.
- SQLAlchemy 2.0 + `asyncpg`.
- Alembic.
- Pydantic Settings для конфигурации.
- `httpx` для Telegram.

Основные сущности:

- `Lead`: имя, телефон, тип проекта, комментарий, статус и дата.
- `Project`: название, факты, описание, порядок, публикация.
- `ProjectMedia`: URL, тип, alt, порядок и связь с проектом.
- `TextSection`: стабильный ключ, заголовок, текст, CTA и порядок.
- `TelegramSettings`: включён/выключен и список admin ID; токен хранится только в окружении.

Отдельная модель пользователя не нужна: в первой версии один администратор.

## API

- `POST /api/leads` — публичная заявка.
- `GET /api/content` — публичный контент и проекты.
- `POST /api/admin/login`, `POST /api/admin/logout` — вход/выход.
- `/api/admin/leads` — список и изменение статуса заявок.
- `/api/admin/texts`, `/projects`, `/media`, `/settings`, `/telegram` — управление сайтом.
- `POST /api/admin/uploads` — безопасная загрузка медиа на VPS.
- `GET /api/health` — готовность приложения и БД.

После сохранения заявки backend пытается отправить Telegram-уведомление. Ошибка Telegram не отменяет запись в БД: заявка остаётся доступной в админке.

## Frontend

- React + TypeScript + Vite.
- Tailwind CSS.
- React Router только для `/`, `/privacy` и `/admin`.
- Native `fetch`; отдельная библиотека состояния не нужна.
- Анимации на CSS и `IntersectionObserver`.
- Контент публичной страницы загружается одним запросом и имеет встроенный fallback для первого рендера.

## Админка

- Одна роль `admin`, один логин.
- Пароль хранится как Argon2-хеш, не в исходниках.
- Сессия — подписанная `HttpOnly`, `Secure`, `SameSite=Strict` cookie.
- Простые таблицы и формы; drag-and-drop допускается только для порядка медиа.
- Все изменения валидируются backend; frontend не является границей доверия.

## Медиа

- Загруженные файлы хранятся на VPS в именованном Docker volume и отдаются по `/uploads`.
- В БД хранятся метаданные и относительный путь.
- Имя файла генерируется сервером; исходное имя не используется как путь.
- Разрешены JPEG, PNG, WebP, AVIF, MP4 и WebM с MIME allowlist и лимитами 15/100 MB.
- При замене файл сначала записывается под новым именем; старый удаляется только после успешной записи БД.
- Объектное хранилище добавляется, когда один VPS перестанет покрывать объём или доступность.

## Безопасность

- Секреты только в `.env` на сервере; `.env` не попадает в Git и образы.
- Все ранее переданные через чаты доступы должны быть заменены до запуска.
- HTTPS обязателен, HTTP перенаправляется на HTTPS.
- CORS закрыт на публичный домен.
- Вход и форма заявки защищены process-local ограничением частоты; форма также имеет honeypot и серверную валидацию.
- Загрузка файлов доступна только после входа; исполняемые расширения запрещены.
- Логи не содержат пароли, токены, cookies и полный текст персональных данных.
- PostgreSQL не публикует порт наружу.
